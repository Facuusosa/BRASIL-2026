# HunterPrice AI - Master Blueprint y Hoja de Ruta de Ejecución

## 1. Misión Inmediata: El Motor de Inteligencia "V1"
El objetivo es consolidar el `engine_precios.py` que transforme el Excel de competencia en un producto de información de alto valor. No buscamos solo datos, buscamos **revelar el ahorro oculto** en la dispersión de precios de los mayoristas.

## 2. Fase 1: Cimientos y Extracción de Valor (Mes 1)
- **Sprint 1: El Unificador Semántico.** Crear el script que realice el "Fuzzy Matching" para unificar los nombres de productos de todos los competidores en una sola base coherente.
- **Sprint 2: El Detective de Precios.** Desarrollar la lógica para identificar automáticamente el "Piso de Mercado" (precio mínimo) y el competidor que lo ofrece.
- **Sprint 3: El Generador de Reportes.** Crear el primer HTML interactivo "Premium" que Facu pueda enviar a un cliente por WhatsApp para demostrar el ahorro posible.

## 3. Fase 2: Validación y Monetización (Mes 2-3)
- **Beta Testing B2B:** Entregar reportes de ahorro gratuitos a 5 comerciantes seleccionados para validar cuánto valor le ven a la información.
- **Canal de Oportunidades (Telegram):** Implementar alertas automáticas de "Flipping" (precios rotos en el mercado) para que Facu capitalice errores de la competencia.
- **Refinamiento de UX:** Ajustar el diseño del HTML según el uso real de los comerciantes en sus depósitos/salones.

## 4. Fase 3: Escalabilidad y SaaS (Objetivo Suiza)
- **Nube y Automatización:** Migrar el motor a un servidor para que el procesamiento sea constante y automático.
- **Plataforma Web:** Pasar del archivo HTML enviado por mensaje a un portal web con suscripción mensual (MRR).
- **Captura Autónoma:** Desarrollar scrapers que alimenten el sistema sin depender de descargas manuales de archivos.

## 5. Estándares de Ejecución para Jarvis (Antigravity)
En cada tarea, Jarvis debe:
1. **Validar contra los 7 MD:** Asegurar que el código no mencione datos internos y respete la visión de negocio.
2. **Documentar Avances:** Actualizar este Blueprint tachando las tareas logradas.
3. **Optimizar el Camino:** Proponer siempre la ruta más rápida hacia un producto que el cliente esté dispuesto a pagar.

## 6. Checklist de Calidad para el Lanzamiento
- [ ] ¿El reporte omite CUALQUIER dato interno (SAP, Vital, Usuarios)?
- [ ] ¿El HTML funciona perfecto en un celular (Mobile-First)?
- [ ] ¿El ahorro detectado es real, comprobable y visualmente impactante?
- [ ] ¿El unificador de nombres (Fuzzy Matching) tiene un error menor al 5%?
